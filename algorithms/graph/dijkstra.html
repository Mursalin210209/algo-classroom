Graph Algorithm

Dijkstra's Algorithm

Intermediate

Find the shortest weighted path in a graph. Uses a priority queue to always expand the lowest-cost unvisited node, guaranteeing optimal distances in graphs with non-negative edge weights.

Time: O((V + E) log V)
Space: O(V)
Shortest path: ✓ Weighted
Structure: Min-Priority Queue
💜
Greedy Choice
Always pick the unvisited node with the smallest known distance — this greedy choice is provably optimal for non-negative weights.
📊
Priority Queue
A min-heap (priority queue) lets us efficiently extract the lowest-cost node in O(log V) time.
🔁
Edge Relaxation
For each neighbor, check if a shorter path exists via the current node. If so, update ("relax") its distance.
⚖️
Non-Negative Weights
Dijkstra requires non-negative edge weights. For negative weights, use Bellman-Ford instead.
