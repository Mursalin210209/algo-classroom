Topological Sort

Intermediate

Order the vertices of a Directed Acyclic Graph so every edge goes left → right. Two classic approaches: Kahn's (BFS / in-degree) and DFS post-order. Both run in O(V+E).

Time: O(V + E)
Space: O(V)
Input: DAG only
Cycle detect: ✓ Both
📐
DAG Prerequisite
Topological sort only exists for Directed Acyclic Graphs. Any cycle makes a linear ordering impossible.
📥
Kahn's — In-degree
Repeatedly remove nodes with in-degree 0 (no prerequisites). Uses a queue; detects cycles if not all nodes are processed.
🌀
DFS Post-order
Run DFS; push each node onto a stack when fully explored (post-order). Pop the stack for topological order.
🎨
3-Color DFS
White = unvisited, Gray = in current path (on stack), Black = finished. A Gray→Gray back edge means a cycle.

💻 Pseudocode — Kahn's
function Kahn(graph): indegree = computeInDegrees(graph) queue = [v for v if indegree[v] == 0] topo = [] while queue not empty: u = queue.dequeue() topo.append(u) for v in graph.adj[u]: indegree[v] -= 1 if indegree[v] == 0: queue.enqueue(v) if topo.length < V: # cycle! return null return topo

function TopoSortDFS(graph): color = {v: WHITE for v in V} stack = [] # result (reversed) for v in V: # visit all if color[v] == WHITE: dfs(v) function dfs(u): color[u] = GRAY # on path for v in adj[u]: if color[v] == GRAY: # cycle! raise CycleError if color[v] == WHITE: dfs(v) color[u] = BLACK # finished stack.push(u) # post-order
