Prim's Algorithm

Intermediate

Grow a Minimum Spanning Tree one edge at a time — always greedily picking the cheapest edge that connects the MST to an unvisited vertex. Uses a min-priority queue for efficiency.

Time: O(E log V)
Space: O(V + E)
Structure: Min-Heap
Result: MST
Strategy: Greedy
🌲
Minimum Spanning Tree
A spanning tree that connects all V vertices with V−1 edges and the minimum possible total edge weight.
🏆
Greedy — Always Cheapest
At each step, Prim's picks the minimum-weight edge that crosses the cut between MST and non-MST vertices.
📉
Min-Priority Queue
A min-heap tracks the cheapest crossing edge for each candidate vertex. Extract-min drives the algorithm.
✂️
Cut Property
The lightest edge crossing any cut of the graph is always safe to include in an MST — the theoretical guarantee behind Prim's.

💻 Pseudocode
function Prim(graph, start): pq = new MinHeap() # (cost, from, to) inMST = new Set() mstEdges = [] pq.push((0, null, start)) while pq not empty: (cost, from, u) = pq.popMin() if u in inMST: continue inMST.add(u) if from is not null: mstEdges.append((from, u, cost)) for (v, w) in graph.adj[u]: if v not in inMST: pq.push((w, u, v)) return mstEdges
