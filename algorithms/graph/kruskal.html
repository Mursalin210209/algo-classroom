Kruskal's Algorithm

Intermediate

Build a Minimum Spanning Tree by sorting all edges by weight and greedily adding the cheapest one that doesn't create a cycle — detected instantly via Union-Find (Disjoint Set Union).

Time: O(E log E)
Space: O(V)
Structure: Union-Find
Cycle detect: O(α(V))
Result: MST
📦
Sort Edges First
Kruskal's pre-sorts all edges by weight ascending. Then scans them in order — the key difference from Prim's vertex-centric approach.
🔗
Union-Find (DSU)
Tracks connected components. Find(u) returns the root; Union(u,v) merges components. Path compression + rank makes it nearly O(1).
🚫
Cycle Detection
An edge (u,v) creates a cycle iff Find(u) == Find(v) — they're already in the same component. Reject it immediately.
✂️
Cut Property
Like Prim's, Kruskal's is correct by the cut property: the minimum-weight crossing edge of any cut is always safe to include in the MST.

💻 Pseudocode
function Kruskal(graph): edges.sortBy(weight) # ascending dsu = DSU(V) # each node = own set mst = [] for (u, v, w) in edges: ru = dsu.find(u) rv = dsu.find(v) if ru == rv: # same component continue # would form cycle dsu.union(ru, rv) mst.append((u, v, w)) if mst.length == V-1: break return mst
