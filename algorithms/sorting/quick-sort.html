Sorting Algorithm
⚡

Quick Sort

Intermediate

Choose a pivot, partition the array so every element left of the pivot is smaller and every element right is larger, then recursively sort each side. In-place and blazingly fast in practice.

Best: O(n log n)
Average: O(n log n)
Worst: O(n²)
Space: O(log n)
Stable: ✗ No
🎯
Pivot Selection
Pick one element as the pivot. Its final position after partitioning is exactly correct — it never moves again.
↔️
Partition
Scan with two pointers: i finds elements ≤ pivot (from left), j scans right-to-left. Swap when they're out of order.
📦
In-Place
Unlike Merge Sort, no extra arrays are needed. All swaps happen within the original array using O(log n) stack space.
📊
Average O(n log n)
With a good pivot, partitions are roughly balanced. Bad pivots (always min/max) degrade to O(n²).
