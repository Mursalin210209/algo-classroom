Sorting Algorithm
⚡

Timsort

Advanced

The hybrid algorithm powering Python's sorted() and Java's Arrays.sort(). Timsort slices the array into fixed-size runs, sorts each with Insertion Sort, then merges adjacent runs in successive passes — combining the best of both algorithms.

Best: O(n)
Average: O(n log n)
Worst: O(n log n)
Space: O(n)
Stable: ✓ Yes
MIN_RUN: 4 (demo)
✂️
Slice into Runs
The array is divided into fixed chunks called runs (MIN_RUN = 32–64 in production; 4 here for clarity).
📌
Insertion Sort
Each run is sorted in-place with Insertion Sort — optimal for tiny subarrays due to low overhead and great cache behaviour.
🔀
Merge Passes
Sorted runs are merged pairwise in log₂(n/MIN_RUN) passes, just like the merge step of Merge Sort.
🌍
Production Born
Used in CPython, PyPy, Java SE, Android, and V8. Beats quicksort on partially-sorted real-world data.

💻 Pseudocode
timsort(arr, MIN_RUN=4): n = arr.length // Phase 1: insertion-sort each run for i=0; i<n; i+=MIN_RUN: end = min(i+MIN_RUN-1, n-1) insertionSort(arr, i, end) // Phase 2: merge sorted runs size = MIN_RUN while size < n: for left=0; left<n; left+=size*2: mid = left + size - 1 right = min(left+2*size-1, n-1) if mid < right: merge(arr, left, mid, right) size *= 2
