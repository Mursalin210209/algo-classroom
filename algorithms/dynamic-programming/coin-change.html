Dynamic Programming

Coin Change Problem

Intermediate

Given coin denominations and a target amount, find the minimum number of coins needed to make up that amount. Each coin can be used unlimited times. If impossible, return −1.

Time: O(amount × coins)
Space: O(amount)
Type: Bottom-up DP
Unbounded: ✓ Yes
🎯
Subproblems
dp[a] = min coins to make amount a. Build from 0 up to target, reusing smaller solutions.
🔄
Recurrence
dp[a] = min over all coins c where c ≤ a of (1 + dp[a − c]). Try every coin at every amount.
♾️
Unbounded Use
Unlike 0/1 Knapsack, each coin denomination can be used as many times as needed.
🔍
Traceback
After filling the table, trace back by repeatedly subtracting the coin that gave the optimal value.

💻 Pseudocode
function coinChange(coins, amount): dp = array of size amount+1 dp.fill(Infinity); dp[0] = 0 for a = 1 to amount: for each coin in coins: if coin ≤ a: candidate = 1 + dp[a - coin] if candidate < dp[a]: dp[a] = candidate return dp[amount] == Inf ? -1 : dp[amount]
