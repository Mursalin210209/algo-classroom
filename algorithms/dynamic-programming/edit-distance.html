Dynamic Programming

Edit Distance (Levenshtein)

Intermediate

Given two strings X and Y, compute the minimum number of edits (insert, delete, substitute) to transform X into Y. Solved with a 2D DP table.

Time: O(m·n)
Space: O(m·n)
Type: Bottom-up DP
Optimal: ✓ Yes
📏
Subproblems
dp[i][j] = min edits for X[0..i-1] to Y[0..j-1]. Build from smaller prefixes.
🔄
Match or Edit
If chars match, no cost (diagonal). Else, cost 1 for insert, delete, or substitute.
📜
Recurrence
dp[i][j] = dp[i-1][j-1] if match, else 1 + min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1]).
🔍
Traceback
Trace back from dp[m][n] to reconstruct the sequence of edit operations.

💻 Pseudocode
function editDistance(X, Y, m, n): dp = (m+1) × (n+1) table for i = 0 to m: dp[i][0] = i for j = 0 to n: dp[0][j] = j for i = 1 to m: for j = 1 to n: if X[i-1] == Y[j-1]: dp[i][j] = dp[i-1][j-1] else: dp[i][j] = 1 + min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1]) return dp[m][n] // traceback for operations
