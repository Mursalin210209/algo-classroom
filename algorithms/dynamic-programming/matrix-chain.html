Dynamic Programming

Matrix Chain Multiplication

Hard

Given a chain of matrices, find the most efficient way to multiply them together. The order of multiplication affects the total number of scalar operations — choose the optimal parenthesization to minimize cost.

Time: O(n³)
Space: O(n²)
Type: Interval DP
Fill order: Diagonal by diagonal
📐
Matrix Cost
Multiplying a (p×q) matrix by a (q×r) matrix costs p×q×r scalar multiplications.
🔗
Interval DP
dp[i][j] = min cost to multiply matrices i…j. Fill by increasing chain length L = 2, 3, …, n.
✂️
Split Point k
Try every k between i and j: cost = dp[i][k] + dp[k+1][j] + p[i-1]×p[k]×p[j]. Take the min.
🔍
Traceback
Store the optimal k in a bracket table. Recursively reconstruct the fully parenthesized expression.

💻 Pseudocode
function matrixChain(p, n): dp[i][i] = 0 for all i for L = 2 to n: // chain length for i = 1 to n-L+1: j = i + L - 1 dp[i][j] = ∞ for k = i to j-1: cost = dp[i][k] + dp[k+1][j] + p[i-1]*p[k]*p[j] if cost < dp[i][j]: dp[i][j] = cost bracket[i][j] = k return dp[1][n], bracket
