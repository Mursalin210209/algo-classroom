Data Structure

AVL Tree

Intermediate

A self-balancing Binary Search Tree where the heights of the two child subtrees of any node differ by at most one. After every insertion, balance factors are checked and rotations are applied to restore balance.

Search: O(log n)
Insert: O(log n)
Delete: O(log n)
Space: O(n)
Always balanced: ✓ Yes
⚖️
Balance Factor
BF = height(left) − height(right). A valid AVL node has BF ∈ {−1, 0, 1}. |BF| ≥ 2 triggers a rotation.
↩️
Right Rotation (LL)
When a node is left-heavy and its left child is also left-heavy. Pivot the unbalanced node down-right.
↪️
Left Rotation (RR)
When a node is right-heavy and its right child is also right-heavy. Pivot the unbalanced node down-left.
🔄
LR / RL Rotations
Double rotations for left-right or right-left imbalances — left-rotate the child first, then right-rotate the parent (or vice-versa).

💻 Pseudocode
function insert(node, val): if node is null: return new Node(val) if val < node.val: node.left = insert(node.left, val) else: node.right = insert(node.right, val) updateHeight(node) bf = height(left) - height(right) if bf > 1 and val < node.left.val: rotateRight(node) # LL if bf < -1 and val > node.right.val: rotateLeft(node) # RR if bf > 1 and val > node.left.val: LR double-rotate # LR if bf < -1 and val < node.right.val: RL double-rotate # RL return node
function search(node, val): if node is null: return false if val == node.val: return true if val < node.val: return search(node.left, val) else: return search(node.right, val)
