Fenwick Tree

Intermediate

Also known as a Binary Indexed Tree (BIT), the Fenwick Tree enables efficient prefix sum queries and point updates on an array. Each index stores a partial sum determined by the lowest set bit (LSB) of its index.

Update: O(log n)
Query: O(log n)
Build: O(n log n)
Space: O(n)
1-Indexed: ✓ Always
📦
Partial Sums
Each cell BIT[i] stores the sum of elements from i − LSB(i) + 1 to i, covering a range determined by the lowest set bit.
⬆️
Point Update
To update index i, propagate changes upward by repeatedly adding LSB(i) to i until out of bounds.
🔢
Prefix Query
Sum A[1..i] by accumulating BIT[i] and moving to i − LSB(i) until i reaches 0.
⚡
LSB Trick
LSB(i) = i & (−i). This bit manipulation is the core of both update and query traversal paths.

💻 Pseudocode
function build(arr[1..n]): BIT = new int[n+1] for i = 1 to n: BIT[i] += arr[i] j = i + LSB(i) if j <= n: BIT[j] += BIT[i]
function query(r): // prefix sum 1..r sum = 0 while r > 0: sum += BIT[r] r -= LSB(r) // r &= r-1 return sum
function update(i, delta): while i <= n: BIT[i] += delta i += LSB(i) // i += i & (-i) return
